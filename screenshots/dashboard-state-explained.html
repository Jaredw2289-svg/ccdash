dashboardState 全解

它是什么、怎么来的、怎么用的 — 一张图说清楚

1. dashboardState 长什么样？
interface DashboardState  ·  src/types/DashboardState.ts
projectLabel string | null 项目名（取自工作目录的 basename，如 "ccdash"）
goalSummary string | null 本次会话的 稳定目标（由 Claude 自己写入，如 "为状态栏添加响应式布局"）
userTurnSummaries string[] 每次用户发消息时记录的摘要列表（按时间顺序）
lastConclusion string | null 最新结论 — Claude 最近一次回复的摘要，或 --now 写入的值
currentFocus string | null 当前焦点 — 通常等于 lastConclusion，回退到最后一条 userTurnSummary
transcriptMtime number | null transcript 文件的最后修改时间（用于判断是否需要重新解析）
updatedAt string | null 最后更新的 ISO 时间戳
一句话总结： dashboardState 是一个 会话级别的摘要对象，记录了「你在做什么」（goal）和「做到哪了」（now/lastConclusion）。 它让状态栏不只显示 token 和 model，还能显示 当前任务语义。
2. 数据从哪来？（三个数据源）
① Claude 主动写入
CLAUDE.md 里指示 Claude 每次回复后调用：
--write-session-status --goal "..." --now "..."
写到 ~/.claude/status/{sessionId}.json
② 用户发消息时
UserPromptSubmit hook 触发
recordUserPromptSummary()
记录用户消息摘要到 userTurnSummaries
③ Transcript 文件
解析 Claude Code 的 JSONL 对话记录
getLastAssistantSummary()
提取最后一条助手回复摘要
3. 完整数据流（从写入到显示）
写入流程（数据怎么存进去）
Claude 回复完毕 → 执行 --write-session-status → writeSessionStatusFile() → ~/.claude/status/{id}.json
写入 goal 和 now 两个字段
用户发消息 → UserPromptSubmit hook → recordUserPromptSummary() → ~/.cache/ccdash/sessions/{id}.json
追加 userTurnSummaries，更新 currentFocus
读取流程（状态栏怎么拿到数据）
Claude Code 管道输入 JSON → ccstatusline.ts (主入口) → getDashboardState(data)
→ → → ↓ 合并三个数据源 → DashboardState → context.dashboardState
→ → → → → ↓
→ → StatusSummaryWidget.render() → "Goal: 为状态栏添加响应式布局" → 终端状态栏 ✨
4. 什么时候更新？（生命周期时间线）
🔵 会话开始
状态文件不存在，getDashboardState() 返回空/null。
状态栏的 StatusSummary widget 什么也不显示。
🟢 用户发出第一条消息
UserPromptSubmit hook 触发 → recordUserPromptSummary()
创建 ~/.cache/ccdash/sessions/{id}.json，写入第一条 userTurnSummary。
{ userTurnSummaries: ["dashboardState 是什么"], currentFocus: "dashboardState 是什么" }
🟡 Claude 回复完毕
Claude 按照 CLAUDE.md 指示执行 --write-session-status，写入 goal 和 now。
同时，transcript 文件被更新。
~/.claude/status/{id}.json → { goal: "理解 dashboardState 原理", now: "已生成原理图" }
🟣 下次状态栏刷新
Claude Code 管道调用 ccstatusline → getDashboardState() 合并：
  1. 读取 ~/.cache/ccdash/sessions/{id}.json（缓存状态）
  2. 读取 ~/.claude/status/{id}.json（Claude 写入的 goal/now）
  3. 检查 transcript 文件的 mtime，按需解析新摘要
  4. 合并后写回缓存
🔴 持续对话中…
每次用户发消息 → 追加 userTurnSummary
每次 Claude 回复 → 更新 goal / now
每次状态栏刷新 → 重新合并三个数据源
goalSummary 一般只在第一轮设定，很少变； lastConclusion 每轮都变。
5. goalSummary 和 lastConclusion 的区别
goalSummary
稳定 会话级

整个会话的 目标。
第一轮由 Claude 写入，之后很少变。

来源：--goal "..."
存储：~/.claude/status/{id}.json

例如："为状态栏添加响应式布局"

显示为：Goal: 为状态栏添加响应式布局
lastConclusion
动态 轮次级

最新的 进度/状态。
每次 Claude 回复后都可能更新。

来源：--now "..." 或 transcript 解析
存储：缓存 + status 文件

例如："已完成 renderer 重构，测试通过"

显示为：Now: 已完成 renderer 重构，测试通过
打比方： goalSummary 像是你的 任务标题（"修复登录页 bug"），一旦定下来就不会变。
lastConclusion 像是你的 实时状态（"正在调试 OAuth 回调"），每做一步就会更新。

两者合在一起，状态栏就能同时告诉你 "你要做什么" 和 "做到哪了"。
6. 存储在哪里？
Session Status File
路径：~/.claude/status/{sessionId}.json

内容：{ goal, now, updatedAt }

Claude 写入 简洁

这是 Claude 通过 --write-session-status 直接写入的文件，只有 goal 和 now 两个语义字段。
Session State Cache
路径：~/.cache/ccdash/sessions/{sessionId}.json

内容：完整的 DashboardState

自动维护 合并后

这是 ccdash 自己维护的缓存，合并了 status file、transcript、用户消息摘要的完整状态。
7. 关键函数一览
核心函数 · src/utils/session-state.ts & session-status-file.ts
getDashboardState() 主入口。合并三个数据源，返回最终的 DashboardState。每次状态栏渲染时调用。
recordUserPromptSummary() 用户发消息时调用。提取消息摘要，追加到 userTurnSummaries。
writeSessionStatusFile() Claude 通过 CLI 写入 goal/now 时调用。写到 ~/.claude/status/。
readSessionStatusFile() 读取 Claude 写入的 status 文件。支持 JSON 和 TXT 两种格式。
refreshStateFromTranscript() 检查 transcript 文件是否有更新（比对 mtime），有的话重新解析最后一条助手摘要。
